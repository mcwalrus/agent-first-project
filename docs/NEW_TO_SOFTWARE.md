# New to Software

This page is for you if you have **little or no background** in making or changing computer programs, but you would like to work with **AI assistants** that can read files, suggest steps, and (in some setups) run actions for you.

**Software** is simply instructions that tell a computer what to do—apps, websites, scripts, and the files that hold them. **Software development** is the work of writing and changing those instructions. You do **not** need to already know that world. This tutorial will introduce terms as they matter; when something sounds technical, check the [Glossary](./GLOSSARY.md) or ask an AI to explain it in everyday language.

You do not need to memorise every tool upfront. It helps to stay curious, to copy error text from your screen when something fails (after removing private details—see below), and to ask often: _what does this mean in plain English?_

The purpose of this tutorial matches [Introduction](./INTRO.md): to help you **see for yourself** whether you can steer an AI to do useful work on real files and tasks. This page only smooths the start; motivation and deeper reading stay in the other documents.

## How to use this tutorial

If words like _repository_ (a folder of project files, often copied from a code-sharing site), _terminal_ (a text window where you type commands), or _backpressure_ (fast, automatic feedback when something breaks) feel unfamiliar, **start here**, then choose a path:

- **Try this sample project on your computer** — When you are ready to install a few tools and run the example app on your machine, go to [Getting Started](./GETTING_STARTED.md). That is the “set up first” path: prepare your environment, install what the project needs, then run the commands it describes (for example `just do-it`).
- **Use an AI that works from a text window with your files** — When you want a tool such as Claude Code that works inside a folder on your computer, follow [Setup Claude Code](./CLAUDE_SETUP.md). That page explains creating an account and running `claude` from the command line.
- **Read ideas without installing everything** — You can read [Introduction](./INTRO.md), [Understanding Your Backpressure](./BACKPRESSURE.md), [Tech Stacks For Agents](./TECH_STACK.md), and [Everything Is A Loop Now](./AGENT_LOOP.md) in any order. When a word appears that you do not know, use the [Glossary](./GLOSSARY.md) or ask an AI to define it.

You do **not** have to finish every setup document in one sitting. Many people alternate: read one short section, try one step, ask an AI what happened, repeat.

## Command line vs Claude Desktop

**Command-line / terminal tools** (for example Claude Code) are built to work inside a **folder of project files** on your computer. They can run commands and edit files in that folder, which fits how this tutorial and [Setup Claude Code](./CLAUDE_SETUP.md) are written.

**Claude Desktop** (or other chat apps) is very good for **explaining** things, talking through what you see on screen, and learning before you automate anything. It is less tied to “this exact folder of files” unless you paste content or add files yourself. Neither option is wrong: use chat when you want clarity; use a command-line assistant when you want the AI to **act** directly on your project files.

## Using AI to explain things

Treat an AI as a patient tutor:

- Ask for **simple explanations** of any specialist word; check the [Glossary](./GLOSSARY.md) when you see the same word again.
- Before you run a command you do not understand, ask what it will do and what might go wrong.
- When you paste text from your screen (for example an error message), **leave out anything private**: passwords; one-time codes; bank or card details; and any long codes a website or service gave you to prove it is really you (those are sometimes called _access codes_ or _API keys_ in other documents—see the glossary when you are ready). If you are not sure whether something is private, replace it with the word `REMOVED` and say you redacted it.
- Ask the AI to turn a confusing paragraph into a short list of steps, or to point out which words are names of tools you could look up later.
